HOW TO PUBLISH FORMA AND HOW TO USE IT FROM MARIMO_LEARN
=========================================================

PART 1: PUBLISHING FORMA TO NPM
--------------------------------

1. Install dependencies and run tests:

       cd ~/forma
       npm install
       npx playwright install chromium   # only needed once
       npm test

   All 91 tests should pass before publishing.

2. Log in to npm (only needed once per machine):

       npm login

   This opens a browser for authentication.

3. Check that the package name "forma" is available:

       npm info forma

   If that returns a 404-style error, the name is free.
   If "forma" is already taken, change the "name" field in package.json
   to something like "@gvwilson/forma" (a scoped name is always available
   under your own npm username).

4. Build the final bundle:

       npm run build

   This produces:
     dist/forma.js          — standalone bundle (the npm package main)
     dist/widgets/          — per-widget ESM files for anywidget use

5. Publish:

       npm publish --access public

   For a scoped package (@gvwilson/forma) the --access public flag is
   required; for an unscoped name it is optional but harmless.

6. Verify the published package:

       npm info forma        # or @gvwilson/forma

   The "dist" tarball should be a few hundred KB.


PART 2: MAKING MARIMO_LEARN USE THE PUBLISHED FORMA PACKAGE
------------------------------------------------------------

Once forma is published on npm, marimo_learn no longer needs to keep its
own copy of the widget JavaScript source. Instead:

Step A — Add forma as a dev dependency of marimo_learn's JS build:

    cd ~/marimo_learn/js
    npm install forma --save-dev

Step B — Update js/build.js to copy forma's per-widget files into the
Python package's static directory instead of building them from source.

Replace the anywidgetConfig esbuild block with a simple file-copy step:

    import fs from 'fs';
    import path from 'path';

    const FORMA_WIDGETS = path.resolve('node_modules/forma/dist/widgets');
    const STATIC_OUT   = path.resolve('../src/marimo_learn/static');

    fs.mkdirSync(STATIC_OUT, { recursive: true });
    for (const file of fs.readdirSync(FORMA_WIDGETS)) {
      fs.copyFileSync(
        path.join(FORMA_WIDGETS, file),
        path.join(STATIC_OUT, file)
      );
    }
    console.log('Copied forma widget files to', STATIC_OUT);

Then build the standalone bundle (dist/marimo-learn.js) from forma's
own bundle rather than rebuilding from source — or simply symlink /
re-export it:

    // bundleConfig in build.js — point to forma's bundle
    import { build } from 'esbuild';
    await build({
      entryPoints: ['node_modules/forma/dist/forma.js'],
      bundle: false,          // already bundled
      format: 'esm',
      outfile: 'dist/marimo-learn.js',
      allowOverwrite: true,
    });

Step C — Remove the js/src/ directory from marimo_learn (it is now
maintained exclusively in the forma repository).

Step D — Update the Makefile so that `make build` runs `npm install`
in js/ before the copy step, ensuring the latest published forma
version is always used:

    build:
        cd js && npm install && node build.js
        uv run python -m build

Step E — Update marimo_learn's CONTRIBUTING.md to note that widget JS
is now maintained in the separate forma repository (~/forma or its
eventual GitHub remote) and that changes must be published to npm
before a new marimo_learn release.


PART 3: WORKFLOW FOR FUTURE WIDGET CHANGES
------------------------------------------

1. Edit source in ~/forma/src/
2. Run `npm test` in ~/forma to verify all 91 tests pass.
3. Bump the version in ~/forma/package.json (follow semver).
4. Run `npm publish --access public` from ~/forma.
5. In ~/marimo_learn/js, run `npm install forma@<new-version>`.
6. Run `make build` in ~/marimo_learn to copy the updated files.
7. Run `make test` in ~/marimo_learn to verify the Python side.
8. Commit and release marimo_learn as usual.
